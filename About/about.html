A little about myself:

Since 2012 I have been answering questions occurring at the cross-roads of hydraulic and environmental engineering. A large portion of this work has been devoted to improving fish passage through engineered hydraulic structures (i.e., roadway culverts and dams).

Both computational fluid dynamics (CFD) and experimental fluid methods have played important roles in my research. In particular, the open-sourced CFD model OpenFOAM, due to its strengths and immense flexibility, has proven a valuable tool in these projects. On the experimental front, stereoscopic particle image velocimetry, which provides three component vectors over a 2-D plane, has allowed me to ‘visualize’ flow fields to levels of spatial and temporal detail that would otherwise have been impossible using other experimental flow measurement techniques.

Aside from my work in fish passage, I also have an interest in all things related to fluid mechanics, hydrotechnical work and fluvial systems. Recently, I compared a two-phase Euler-Euler computational fluid dynamics model to a comprehensive laboratory data set with the goal of assessing the model’s ability to model bubble driven recirculatory flow in a municipal waste-water treatment plant. I was also involved in validation work of the Vectrino Profiler performed at the Université de Sherbrooke. Soon, I will be embarking on a project using OpenFOAM to model suspended sediment transport loads through a municipal sediment trap near the mouth of river located in Southern Québec. Have a look at my Research projects page for in-depth looks at some of my work.

My plan is to continue pursuing projects, either in academia or in industry, with a strong emphasis on the application of computational fluid dynamics and experimental methods to address fundamental and applied questions in the field of ecohydraulics.

Computational fluid dynamics

I took an interest in computational fluid dynamic (CFD) modeling in late 2012 starting with an undergraduate project modeling shear-stress distributions in a river bend using a 2D model CCHE2D. Ever since, I have been progressively improving my pratical skills and theoretical knowledge related to using CFD to investigate research and industry projects in 2D and 3D. After working for two years with Flow-3D, I began my journey using OpenFOAM to model free-surface, bubbly and sediment laden flows. The low-level nature of OpenFOAM forced me to learn a great deal about many of the technicalities of CFD which users of most commercial CFD software are generally not exposed. My experience with OpenFOAM over the past four years has piqued my interest in the complicated world of the numerical methods employed in CFD. In early 2019, I began manually translating uFVM (a Matlab finite-volume CFD code) into Python to learn every aspect of the numerical methods employed in a modern finite-volume method CFD solver such as OpenFOAM. The source code of this translation, called pyFVM is available on github and is expected to be fully functional in mid 2020.

CFD models I currently use:

  • OpenFOAM (interFoam, twoPhaseEulerFoam, sedFoam, other solvers)
  • Telemac2D (2D hydrodynamic model with sediment, tracer, thermal and pollutant transport capabilities)
  • FLOW-3D (free-surface 3D hydrotechnical CFD model)

Programming

Python is my main programming language. Early in my research career, I chose to focus my efforts on learning Python because it can do anything Matlab can but it is free, it can run on anything, it has strong object-orientated programming capabilities and is it supported by an ever growing community of researchers in pretty much every field from astrophysics to nanotechnology.

Through the years, aside from using Python to perform routine data processing, analysis and postprocessing, I have also taken on a number of larger scale object-orientated projects with the language. During my PhD I developed pyTracker3D which allows users to reconstruct the 3D trajectories of moving objects by passing the centroids of the objects tracked in at least two views through a direct linear transformation algorithm. This project enabled me to gain a considerable amount of experience with computer vision, a field which persists to interest me in my ongoing research project with tracking the mobile bed layer of sediment. The source code for pyTracker3D is available on Github.

In 2019 I used Python to develop blockMeshBuilder (also available on Github), which through the use of VTK bindings, allows the user to interactively construct very complicated multiblock meshes. The user first populates a Python scipt with the number of blocks needed in the x, y and z directions, then they can interactively remove the unnecessary blocks to arrive at the desired multiblock mesh configuration. Boundary patch groupings are also interactively selected and labelled. The blockMesh file is automatically populated with all the correct face numbers and vertices ordered accurately. I used blockMeshBuilder in a recent project to build a mesh containing 30+ blocks in only 9 lines of code.

The translation of uFVM (a Matlab finite-volume method solver) into Python which I mentioned above, has been by far the project on which I have worked that makes the most comprehensive use of Python's object-orientated capabilities. Just to give an idea of scale, so far the pyFVM codebase has 8379 lines of code. At its current state the code is generalized in a way to read in any unstructured finite-volume method mesh following OpenFOAM's mesh format. The numerics are still under-developement, but pyFVM can calculate the gradient for any scalar field variable over any size of OpenFOAM mesh.
